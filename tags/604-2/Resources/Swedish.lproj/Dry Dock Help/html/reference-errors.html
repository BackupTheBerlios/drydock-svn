Fel och varningar

Många kommandon i Torrdockan kan ge upphov till fel, varningar eller anmärkningar, som presenteras med dialogrutan “Problemrapport” som visas härunder.

Figur 1. En typisk Problemrapport-dialogruta.

Denna hjälpsida presenterar exempel på de vanligaste felen som kan uppstå, samt förklarar varför de inträffar.

Om hjälpsidan inte ger tillräcklig information för att lösa problemet, kan du välja att skicka en felrapport eller be om hjälp via e-post. Notera i så fall att du kan markera ett eller flera meddelanden i dialogrutan Problemrapport och kopiera dem (med kommandot Kopiera i Redigera-menyn).

Varningar och anmärkningar

Dokumentet innehåller icke-triangulära sidor. För att spara det i det valda formatet kommer Torrdockan att omvandla det till trianglar.

Meddelandet framförs när du sparar i Oolite DAT-format, om dokumentet innehåller polygon med fler än tre punkter. On du väljer att fortsätta kommer dokumentet omvandlas till trianglar innan det sparas. Detta motsvarar kommandot Omvandla till trianglar i menyn Verktyg.

Ingen textur vid namn ”Min textur” kunde hittas; alternativtextur används i stället.

När ett dokument öppnas, letar Torrdockan efter texturer på flera platser:

  1. I samma mapp som dokumentet.
  2. I en mapp vid namn ”Textures” i samma mapp som dokumentet.
  3. I en mapp vid namn ”Textures” i den mapp, som innehåller mappen med dokumentet.
  4. Inuti programmet Oolite, om det kan finnas.

Om en textur inte kan hittas i någon av dessa platser visas ovanstående meddelande, och Torrdockan använder alternativtexturen, som är ett grått schackrutemönster.

Figur 2. Alternativtexturen.

Notera att Torrdockan minns texturens namn, och om du sparar dokumentet och öppnar det igen kommer Torrdockan än en gång leta efter den rätta texturen. Med andra ord skadas inte dokumentet utan visas bara felaktigt.

Dokumentet innehåller flera hänvisningar till materialbibliotek. I nuläget stöder Torrdockan endast ett materialbibliotek. Hänvisningen till materialbiblioteket ”Andra materialbibliotek” ignoreras.

Lightwave OBJ-dokument anger materialegenskaper, som texturer, i externa filer som kallas ”materialbibliotek” (och som använder filändelsen ”.mtl”). Varje OBJ-dokument kan hänvisa till noll eller fler materialbibliotek. Torrdockan stöder dock i nuläget endast ett materialbibliotek. Meddelandet ovan visas därför om du försöker öppna ett OBJ-dokument som hänvisar till flera. Det går bra att öppna dokumentet, men vissa texturer kan komma att saknas. Det verkar inte troligt att denna begränsning blir ett problem när man arbetar med modeller för Oolite, men om det visar sig vara det, kan du begära åtgärd via e-post.

Dokumentet innehåller rader av typ ”cstype” (typ av parametrisk kurva/yta), vilka kommer att ignoreras.

Lightwave OBJ-dokument kan innehålla diverse olika typer av geometrisk data som Torrdockan i nuläget inte kan hantera, såsom välvda ytor och hänvisningar till andra OBJ-filer. Om du behöver stöd för en viss funktion, kan du begära detta via e-post. Bifoga gärna exempelfiler.

Texturen ”Min textur” har mått som inte är tvåpotenser. Den har skalats om från 200 × 200 bildpunkter till 256 × 256 bildpunkter.

Som Oolite (och de flesta 3D-spel), kräver Torrdockan att texturer har sidor vars längd i bildpunkter (pixlar) är en tvåpotens. Liksom Oolite kan Torrdockan skala om texturer om nödvändigt för att uppnå detta. Att skala upp texturer slösar dock på minne (både vanligt minne och grafikminne), så texturer bör ändras så att detta inte behövs. För Oolite rekommenderas att texturer över 512 × 512 bildpunkter inte används, och att 256 × 256 punkter används om möjligt. Notera att två texturer på 256 × 256 punkter tillsammans använder hälften så mycket minne som en 512 × 512-textur!

Notera också att olika grafikkort har olika storleksbegränsningar. Torrdockan skalar om nödvändigt ner texturer till den största storlek som anges av grafikkortet. Oolite saknar dock för närvarande denna funktion.

Felmeddelanden

En minnestilldelning misslyckades. Detta beror förmodligen på minnesbrist.
Texturen ”Min textur” kunde inte läsas in, eftersom det fanns för lite minne.

Meddelandet innebär att Torrdockan bad systemet om minne, men inget kunde tillhandahållas. Detta innebär oftast att både det fysiska minnet och huvudhårddisken (startskivan) är fulla. Du bör skapa utrymme på skivan genom att avlägsna onödiga filer och tömma Papperskorgen, samt frigöra minne genom att stänga dokument och program.

Ingen data kunde läsas från ”Mitt dokument”.

Meddelandet innebär att inläsningen av ett dokument misslyckades. Detta kan hända av diverse olika anledningar, som att filen försvinner innan Torrdockan försöker öppna den, eller att skivan med filen är skadad. I allmänhet betyder det inte att det är fel på filens innehåll. Beroende på det specifika problemet kan ett ytterligare förtydligande anges.

Punktdata kunde inte läsas för punkt 37.
Polygon 10 anger punktindex 25, men det finns bara 24 punkter i dokumentet.
Ogiltigt punkttal (2) för polygon 84. Varje sida måste ha minst tre och högst 16 punkter.
Tolkningsfel på rad 56: väntade heltal, fann filslut.

Dessa felmeddelanden beror på olika fel som kan påträffas vid läsning av Oolite DAT- eller Lightwave OBJ-dokument. Såvida du inte redigerat filen i fråga i en textredigerare är skillnaden mellan de olika felmeddelandena oviktig; alla innebär att dokumentet är ogiltigt.

Dokumentet innehåller 602 polygon, men det valda filformatet stöder inte fler än 512.

Meddelandet visas om du försöker spara ett dokument i Oolite DAT-format som är för avancerat för att Oolite skall kunna öppna det. Du måste minska komplexiteten genom att förenkla modellen eller bryta den i delar som sammanfogas genom underenhetssystemet.

Dokumentet innehåller en textur vid namn ”Min textur”. Det valda filformatet tillåter inte texturnamn som innehåller mellanslag, kommatecken, radbrytningar, ”#” eller ”//”.

Detta fel kan uppstå när du försöker spara ett dokument i Oolite DAT-format. Formatet har begränsningar på möjliga filnamn för texturer, vilka klargörs i meddelandet. För att undvika problem rekommenderas filnamn som består av små bokstäver, siffror, bindestreck och understrykningsstreck.

Dokumentet kunde inte sparas.
Dokumentets materialbibliotek kunde inte sparas.

Dessa felmeddelanden innebär att något gick fel när ett dokument skulle skrivas till en skiva, till exempel för att du inte har behörighet att skriva filer till den angivna platsen eller för att skivan är full. Det innebär inte något problem med att skapa filens innehåll; det bör vara möjligt att spara dokumentet på en annan plats eller en annan skiva. Beroende på det specifika problemet kan ett ytterligare förtydligande anges.

Texturen ”Min textur” kunde inte läsas in, eftersom ett fel av typ -42 uppstod.

Detta felmeddelande innebär att en textur inte kunde läsas in, vilket kan ske av diverse olika anledningar, som att filen försvinner innan Torrdockan försöker öppna den, eller att texturen är av okänd typ. (Torrdockan använder Quicktime för att läsa in texturer och stöder därmed ett stort antal format. Det rekommenderade filformatet för texturer är dock PNG.)

Dokumentet är inte ett giltigt Torrdockan-dokument.
Dokumentet förefaller komma från en senare version av Torrdockan; det har ett inkompatibelt format.

Dessa felmeddelanden innebär att filen ser ut som ett Torrdockan-dokument, men inte kan läsas som ett sådant.

Dokumentet är för komplext för att läsas in av Torrdockan. Torrdockan kan inte hantera modeller med fler än 65534 polygon, men dokumentet innehåller 100273.

Torrdockan är avsedd för relativt enkla modeller. Oolite klarar inte att läsa in modeller med fler än 320 punkter, 512 polygon eller 8 material. Torrdockan utökar begränsningen något till 65534 av varje. Notera att flera olika modeller kan sammanfogas genom systemet med underenheter.
